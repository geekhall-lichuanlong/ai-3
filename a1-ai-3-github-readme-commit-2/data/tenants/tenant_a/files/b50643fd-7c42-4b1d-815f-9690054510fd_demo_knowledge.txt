Tenant A 企业知识库测试文档

一、服务等级协议
Tenant A 的生产系统月度可用性目标是 99.95%。如果月度可用性低于 99.9%，客户可以申请服务补偿。P1 事故必须在 10 分钟内响应，并在 30 分钟内给出初步处置方案。

二、数据隔离策略
Tenant A 的所有客户数据必须存储在独立租户空间中。任何查询、检索、导出操作都只能访问 tenant_a 自己的知识库和索引。系统不允许通过请求参数切换到其他租户的数据。

三、AI 模型偏好
Tenant A 默认优先使用低延迟模型回答知识库问题。当问题涉及合规、合同或安全策略时，需要使用更高质量模型，并在回答中附带来源文件和相关段落。

四、访问配额
Tenant A 每日默认 token 配额为 2000。超过配额后，系统必须返回清晰错误信息，不能静默失败，也不能继续调用大模型产生额外费用。

五、文件摄取规则
Tenant A 支持上传 PDF、Markdown 和 TXT 文件。上传后的文件需要先分块，再写入当前租户独立索引。推荐 chunk_size 为 500，chunk_overlap 为 80，以兼顾语义完整性和检索精度。

六、审批流程
当客户申请导出超过 100MB 的数据时，需要管理员审批。涉及个人信息的数据导出还需要安全负责人二次确认。审批记录至少保留 180 天。

七、推荐测试问题
1. Tenant A 的 SLA 是多少？
2. P1 事故需要多久响应？
3. Tenant A 的 token 配额是多少？
4. 为什么不能跨租户检索？
5. 大文件导出需要谁审批？
